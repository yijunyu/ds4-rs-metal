# ds4_metal — Metal shader provenance

ds4_metal's runtime MSL is assembled in `build.rs` from **two distinct sources**
with different copyright/provenance. This file marks which is which, so a public
release correctly attributes the antirez-derived portions (see `NOTICE`).

## 1. antirez bridge — VERBATIM UPSTREAM (antirez/ds4, MIT)

`build.rs` `build_antirez_bridge()` concatenates the upstream antirez Metal
shaders, in antirez's own `required_sources` order, with only the kernel entry
names rewritten (`host_name("kernel_*")` → `ds4_kernel_*`) — the kernel *bodies*
are unchanged antirez source.

- **Source tree:** `benchmarks/ds4_msl/upstream/ds4/metal/*.metal`
  (located via `$DS4_UPSTREAM_DIR`, the workspace `benchmarks/ds4_msl/upstream/`,
  or the sibling-repo fallback — see `build.rs` `locate_upstream()`).
- **Files (antirez order):** `flash_attn.metal`, `dense.metal`, `moe.metal`,
  `dsv4_hc.metal`, `unary.metal`, `dsv4_kv.metal`, `dsv4_rope.metal`,
  `dsv4_misc.metal`, `argsort.metal`, `cpy.metal`, `concat.metal`, … (the full
  `required_sources` list in `build.rs`).
- **Upstream pin:** the antirez C engine FFI (prefill path) is sha-pinned at
  antirez/ds4 commit **`d615ab0`**; the bridge MSL is taken from the same
  upstream checkout. Pin the metal source to that commit when vendoring for a
  public release.
- **Copyright:** © The ds4.c authors / © The ggml authors, MIT (see `NOTICE`).
  These shaders MUST carry the upstream MIT notice in any redistribution.

## 2. tile-rs-generated — ORIGINAL (tile-rs codegen output)

`build.rs` also concatenates the kernels emitted by **tile-rs's `metal` backend**
(the `mlir_to_msl` emitter, i.e. `TILERS_CODEGEN_PATH=metal`). These are codegen
output, not antirez source.

- **Source tree:** `benchmarks/ds4_msl/emitted/*.metal`
  (e.g. `argsort_f32_i32_desc*.metal`, `argsort_merge_*.metal`, …).
- **Origin:** generated by tile-rs (https://github.com/yijunyu/tile-rs) — the
  Metal target of the framework. Regenerable from the tile_std kernel sources.
- **Copyright:** the ds4_metal / tile-rs authors, MIT OR Apache-2.0.

## 3. Hand-written ds4_metal kernels — ORIGINAL

Kernels authored directly in this crate (e.g. the GLM-5.2 Metal kernels
`ds4_kernel_glm_*`, `crates/ds4_metal/patches/*.metal`, and the inline-MSL
kernels in `src/`) are original work, MIT OR Apache-2.0.

---

**Rule for OSS release:** keep the antirez upstream `metal/*.metal` (group 1)
under their MIT notice (`NOTICE`); groups 2 and 3 are ours. A `git grep`-able
marker is the `ds4_kernel_*` rename (group 1) vs the `emitted/` path (group 2).
